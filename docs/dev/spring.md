# 2026-07-03 | @Transactional annotation
- - -
Аннотация **@Transactional** нужна для того, чтобы метод выполнялся в рамках транзакции базы данных. Но чтобы правильно ей пользоваться, нужно понимать как она работает под капотом.

Для начала нужно понять, что при если ты указываешь аннотацию над классом или методом, то бин будет обёрнут в прокси(по умолчанию **CGLIB**). Важный момент!!! Если ты поставишь несколько аннотаций **@Transactional** над разными методами, то это означает, что бин будет обёрнут сразу в несколько прокси. Если бы это было так, то это убило бы всю производительность. Поэтому создаётся один прокси, который и перехватывает вызовы.

Также нужно понимать, что если вызвать обычный метод и внутри него вызвать метод, который помечен как **@Transactional**, прокси не перехватит вызов. Если требуется такая логика, то тут либо устанавливать аннотацию **@Transactional** над первым методом, либо делать **self-inject**.

```java
public class ChatService {
    
    public void methodA() {
        // TODO
        methodB();
    }

    @Transactional
    public void methodB() {
        // TODO
    }
}
```

Есть несколько важных параметров, которые нужно уметь настраивать:

- **propagation** - указывает, что делать из транзакционного метода был вызван другой траInfrastructureProнзакционный метод. По умолчанию ничего не происходит и действие происходит во внешней транзакции, но это поведение можно переопределить.
- **isolation** - устанавливает уровень изоляции транзакции(как в БД). То есть определяет, какие данные будут видны другим параллельным транзакциям.
- **rollbackFor/noRollbackFor** - позволяет указать exception'ы, при возникновении которых нужно откатывать или сохранять транзакцию.
