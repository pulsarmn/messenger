# 2026-07-02 | Repository management
- - -
Стратегии именования веток в проекте:
- **feature/** - новая функциональность, улучшения кода
- **fix/ | hotfix/** - исправления ошибок, багов
- **refactor/** - рефакторинг кода
- **docs/** - написание документации

Стратегии именования коммитов:
- **feat** - новая функциональность, улучшение
- **fix** - исправление ошибки
- **refactor** - рефакторинг кода (без изменения функционала)
- **docs** - только документация
- **test** - добавлены или изменены тесты
- **build** - изменения в сборке проекта
- **chore** - всё остальное(dependencies, gradle, etc)
- **style** - изменение стиля кода
- **revert** - отмена коммита

Структура коммита:
```text
# <тип>(<область>): <краткое описание>
#
# Подробное описание (если нужно)
# - Что было сделано
# - Зачем это сделано
# - Какую проблему решает
#
# Ссылки:
# - Closes #123
# - Fixes #456
```
