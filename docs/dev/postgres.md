# 2026-07-02 | PostgreSQL 18 in Docker
- - -
Начиная с версии **PostgreSQL 18** появилось изменение, которое ломает все прошлые контейнеры **postgres**. А именно изменилась структура хранения данных.

Раньше данные хранились в директории **/var/lib/postgresql/data** и мы связывали эту директорию контейнера с **Docker** **volume**. Теперь же решили хранить данные по другому. Чтобы упростить миграцию на новые версии **PostgreSQL** решили, что структура будет выглядеть вот так **/var/lib/postgresql/<major>/data**, то есть будет одна промежуточная директория, обозначающая **major** версию бд.

Больше можешь узнать тут: https://aronschueler.de/blog/2025/10/30/fixing-postgres-18-docker-compose-startup/


# 2026-07-02 | BIGSERIAL vs BIGINIT GENERATED
- - -
Начиная с какой-то там версии **PostgreSQL** (вроде 10), использование типа **BIGSERIAL** считается устаревшим. Вместо этого теперь нужно использовать следующий синтаксис:

```sql
CREATE TABLE tests (
    id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY
    -- TODO
);
```

Под капотом всё также создаётся отдельный **SEQUENCE** объект, но разница состоит в том, что в данном случае нельзя будет вставить собственное значение для **id**, а в **BIGSERIAL** можно. 

К тому же **BIGSERIAL** - это чисто синтаксис **PostgreSQL**, в других СУБД такого нет. А вот синтаксис **BIGINT GENERATED ALWAYS AS IDENTITY** это стандарт **SQL**.
